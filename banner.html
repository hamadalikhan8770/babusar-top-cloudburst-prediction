📋 Project Overview

Objective: Develop a deep learning model to predict extreme rainfall events (cloudbursts) 2 hours in advance for Babusar Top, Kaghan Valley, Pakistan.

Why This Matters: Cloudbursts cause flash floods, landslides, and infrastructure damage. Traditional weather models fail in complex mountain terrain. This project fills that critical gap with AI-driven localized prediction.

📍 Location Details

Coordinates 35.6°N, 73.6°E
Region Kaghan Valley, Khyber Pakhtunkhwa, Pakistan
Altitude ~4,173 meters above sea level
Terrain Complex mountain geography with steep slopes

🎯 Key Metrics

Cloudburst Definition ≥20 mm/hr or 95th percentile rainfall (whichever is greater)
Prediction Window 2-hour advance lead time for actionable warning
Class Imbalance Only 1.23% cloudburst events (764 out of 61,998)
Data Coverage 2014–2023 (10 years) with 2018-2019 gaps

🧠 Model Architecture

Why BiLSTM?

Bidirectional LSTMs process sequences in both forward and backward directions, capturing temporal patterns that unidirectional models miss. For atmospheric sequences, this means detecting both leading indicators and overall patterns in a 6-hour window.

Final Architecture (171,105 parameters):
  • Input: (6 timesteps × 26 features)
  • BiLSTM Layer: 128 units × 2 directions = 256 total [158,720 params]
  • Dense Layer: 48 units + ReLU activation + L2 regularization [12,336 params]
  • Dropout: 50% during training
  • Output: Sigmoid activation → cloudburst probability (0-1) [49 params]

Handling Extreme Class Imbalance (1.23% events)

Focal Loss
Down-weights easy negatives, focuses on hard positives
Gamma (γ) = 2.0
Alpha (α) = 0.5
Balanced Class Weights
Sklearn's compute_class_weight('balanced')
Weight (Positive) ≈ 40.7
Weight (Negative) ≈ 0.505
Result: Model pays 80× more attention to rare cloudburst events, preventing it from collapsing into a "always predict no cloudburst" trivial solution.

🎓 Training Strategy

Rolling Window Cross-Validation

Instead of random splits (which leak future information into training), we use temporal rolling validation to simulate real-world deployment:

Window Train Years Validation Years Test Set
1 — 2014, 2015 Baseline only
2 2014–2015 2016–2017 —
3 2014–2017 2020 —
4 2014–2020 2021 —
5 2014–2021 2022 —
Final Test Set: 2023 (unseen, held out)
Note: Years 2018–2019 missing from dataset (data availability issue)

Hyperparameter Tuning

Method: 30 trials of random search with rolling window cross-validation

Hyperparameter Search Space Best Value
LSTM Units 32, 64, 128 128
Dropout Rate 0.2, 0.3, 0.4, 0.5 0.50
Dense Units 16, 32, 48, 64 48
Learning Rate 1e-4 to 1e-2 0.005
Optimizer Adam Adam
Best Result (Trial 14): Average validation PR-AUC = 0.4375 across all rolling windows

⏱️ Compute Time: ~3 hours for 30 trials on standard GPU. Metric: PR-AUC optimized for imbalanced classification (ROC-AUC is misleading for rare events).

✨ Key Innovations

  • Temporal Validation: Rolling window CV prevents data leakage and simulates real forecasting deployment
  • Focal Loss: Handles extreme class imbalance (1.23% events) without collapsing to trivial predictions
  • Cyclical Time Features: Sine/cosine encoding captures monsoon seasonality without discontinuities
  • BiLSTM Architecture: Bidirectional processing captures both leading indicators and overall patterns
  • Mountain-Specific: Designed for Babusar Top's complex terrain where traditional NWP models fail
  • 2-Hour Lead Time: Operationally useful for community evacuation and early warnings
  • Production-Ready: Containerized API with authentication, rate limiting, and live data integration
